# MCPScan First 10 Customers

Use this list to find the first 10 paid audit buyers without paid prospecting tools. The goal is not a perfect total addressable market map. The goal is 10 concrete conversations with teams that already have MCP or agent infrastructure touching real systems.

## Qualification Rule

A first-10 prospect is worth outreach when at least 3 of these are true:

- They mention MCP, agents, Claude Desktop, Cursor, internal AI tools, tool-calling, or automation workflows publicly.
- They sell to businesses, not only consumers.
- Their product can touch source code, tickets, docs, CRM, databases, cloud accounts, customer data, or internal APIs.
- They are preparing enterprise pilots, security reviews, procurement, or customer rollouts.
- The likely buyer is reachable directly: founder, CTO, head of engineering, AI lead, security lead, or agency owner.
- They are small enough to buy a fixed-scope service quickly: roughly 5-200 employees.

Avoid prospects where the only visible MCP usage is a toy repo, a personal experiment, or an open-source hobby project with no company buyer.

## Exact First 10 Profiles

### 1. Developer Tool Startup Adding MCP

- Buyer: founder, CTO, head of product, or developer relations lead.
- Trigger: announces an MCP server, agent integration, IDE integration, or "connect your tools to AI" feature.
- Pain: enterprise users will ask what the integration can access.
- Offer: MCP Launch Audit.
- Search terms: `"MCP server" "developer tools"`, `"Model Context Protocol" "startup"`, `"Claude" "MCP" "GitHub" "YC"`.
- Message angle: "Before customers ask, can you show what the MCP integration can access?"

### 2. AI Coding Tool or Code Review Assistant

- Buyer: CTO, security lead, founder.
- Trigger: product reads repositories, pull requests, CI, issue trackers, or local code context.
- Pain: source code access creates immediate security-review friction.
- Offer: MCP Enterprise Readiness Audit if selling to larger companies, otherwise Launch Audit.
- Search terms: `"MCP" "code review"`, `"AI coding agent" "MCP"`, `"Cursor" "MCP" "repository"`.
- Message angle: "Your agent likely sits close to code and secrets; we can map the MCP surface before a customer review."

### 3. AI Agent Agency Building Client Workflows

- Buyer: agency founder, solutions lead, automation lead.
- Trigger: publishes case studies around AI agents, ops automation, custom internal tools, or "AI employee" workflows.
- Pain: client handoff needs a simple security artifact.
- Offer: white-label friendly Quick Audit or Launch Audit.
- Search terms: `"AI agent agency" "MCP"`, `"AI automation agency" "Claude"`, `"custom AI workflows" "MCP"`.
- Message angle: "Give clients a short answer to what the agent can access before handoff."

### 4. B2B SaaS Startup Running Internal AI Agents

- Buyer: CTO, VP engineering, AI platform lead.
- Trigger: posts about internal support agents, sales ops agents, data agents, or agentic workflows.
- Pain: agents may touch customer tickets, CRM, docs, or production data.
- Offer: Launch Audit.
- Search terms: `"internal AI agent" "MCP"`, `"Claude Desktop" "company" "MCP"`, `"Cursor" "internal tools" "MCP"`.
- Message angle: "Make the internal rollout safer before more teams connect credentials."

### 5. Data or Analytics Startup With Agent Connectors

- Buyer: CTO, head of data, security lead.
- Trigger: AI analyst, database assistant, warehouse connector, or natural-language BI workflow.
- Pain: database and warehouse access raises high-severity permission risk.
- Offer: Enterprise Readiness Audit when production data is in scope.
- Search terms: `"MCP" "database" "AI analyst"`, `"MCP server" "Postgres" company`, `"AI data assistant" "MCP"`.
- Message angle: "Which tools can query data, export data, or reveal credentials?"

### 6. Support or Customer Success AI Tool

- Buyer: founder, CTO, security lead.
- Trigger: connects agents to Zendesk, Intercom, Slack, Notion, Linear, Jira, or CRM.
- Pain: customer conversations and internal notes create privacy and trust questions.
- Offer: Launch Audit.
- Search terms: `"MCP" "Zendesk"`, `"AI support agent" "MCP"`, `"MCP" "Intercom"`.
- Message angle: "Review the customer-data paths before a pilot expands."

### 7. Security Startup Experimenting With Agent Workflows

- Buyer: founder, CTO, head of research.
- Trigger: posts about AI SOC, agentic security workflows, automated triage, tool-calling, or MCP connectors.
- Pain: security tooling often has broad access to logs, cloud accounts, tickets, and incidents.
- Offer: Enterprise Readiness Audit.
- Search terms: `"MCP" "security" "agent"`, `"AI SOC" "MCP"`, `"agentic security" "MCP"`.
- Message angle: "Security buyers will ask detailed access questions; the audit gives you a concrete answer."

### 8. Open-Source MCP Server Maintainer With a Company Behind It

- Buyer: maintainer-founder, DevRel, CTO.
- Trigger: popular MCP server repo maintained by a commercial product or consultancy.
- Pain: trust and adoption depend on clear access boundaries.
- Offer: Quick Audit for one server or Launch Audit for a small suite.
- Search terms: `site:github.com "mcp-server" "company"`, `site:github.com "Model Context Protocol" "npm"`.
- Message angle: "A third-party review can make the server easier for teams to approve."

### 9. Startup Preparing Enterprise Pilot

- Buyer: founder, CTO, sales engineer.
- Trigger: hiring enterprise sales, announcing design partners, SOC 2 work, procurement asks, or security questionnaires.
- Pain: MCP and agent access can become a blocker during vendor review.
- Offer: Enterprise Readiness Audit.
- Search terms: `"MCP" "enterprise pilot"`, `"AI agent" "SOC 2" "MCP"`, `"security questionnaire" "MCP"`.
- Message angle: "Turn the MCP setup into a reviewer-friendly artifact before procurement asks."

### 10. Internal Platform Team at a Mid-Market Company

- Buyer: head of platform, principal engineer, security engineer, AI enablement lead.
- Trigger: public engineering post about internal AI tooling or MCP adoption.
- Pain: internal adoption can spread faster than central security review.
- Offer: Launch Audit.
- Search terms: `"internal AI tools" "MCP"`, `"engineering blog" "Model Context Protocol"`, `"Claude Desktop" "engineering blog"`.
- Message angle: "Create a baseline inventory and fix list before the pattern spreads."

## First 10 Booking Targets

Use this booking mix:

- 3 developer tool or AI coding startups.
- 2 AI agent agencies.
- 2 B2B SaaS teams using internal agents.
- 1 data/analytics agent company.
- 1 security agent startup.
- 1 open-source MCP maintainer with commercial backing.

Expected conversion from a cold but targeted list:

- 100 sourced accounts.
- 60 contacted by email or LinkedIn.
- 15 follow-up conversations.
- 5 serious replies.
- 2-3 paid audits.

To reach 10 customers, run the 100-target cycle 3-4 times and improve targeting after every batch.
