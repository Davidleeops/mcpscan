Delivery Console

Use this after payment clears and intake is complete. It turns the paid audit into a repeatable workspace, report, and delivery packet.

Customer Data Rule

Copy the workspace template outside the public repository before adding customer-specific material. Never store customer secrets, private configs, customer data, or final private reports in the public MCPScan repo.

Workspace Setup

Create a private folder for each paid audit, then copy the template into it.

Open template 
mkdir -p "$HOME/MCPScan Audits/YYYY-MM-DD_customer-name"
cp -R delivery/customer-workspace-template "$HOME/MCPScan Audits/YYYY-MM-DD_customer-name/customer-workspace"

1. Intake

Confirm scope, owners, secure handoff path, and sanitized materials.

Intake checklist Secure handoff

2. Scan

Run MCPScan against sanitized configs and save output into the private workspace.

npm run build
node packages/cli/dist/index.js scan PATH_TO_SANITIZED_CONFIG --format json

3. Findings

Track every customer-visible issue with severity, evidence, risk, recommendation, owner, and due date.

Findings tracker

4. Report

Produce a readiness decision: approve, approve with guardrails, or block until remediation.

Report template Cover note Customer comms

Delivery Quality Check

  • Every high or medium finding has evidence and a specific remediation step.
  • The report states a clear readiness decision.
  • The report does not expose secrets or customer data.
  • The customer knows whether a findings call or re-scan is included.
  • The final report is stored outside the public repo.
