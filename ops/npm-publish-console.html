npm Publish Console

Use this only after the repo is green and you are logged into npm with publish rights. npm may require browser login or a one-time password.

Open npm login

Publish Gate

Do not publish until npm whoami succeeds for the intended npm account and you are ready to approve 2FA/OTP prompts. Package publishes are public and version numbers cannot be reused.

1. Login Check

Run this first. If it fails, complete npm login in the browser or terminal.

npm whoami

2. Full Preflight

This runs auth check, package-name checks, install, build, typecheck, tests, audit, and dry-run packs.

npm run release:preflight

3. Dry-Run Packs

Use these if you want to inspect each package tarball output separately before publishing.

npm pack --dry-run -w @mcpscan/shared
npm pack --dry-run -w mcpscan

4. Publish Order

Publish shared first, then the CLI. If npm asks for an OTP, enter the current code from the npm account authenticator.

npm publish -w @mcpscan/shared --access public
npm publish -w mcpscan --access public

After Publish

Verify the public install path, then add the npm package URL to the launch docs and outbound copy only after the package pages are live.

npm view @mcpscan/shared name version
npm view mcpscan name version
npx mcpscan --help
