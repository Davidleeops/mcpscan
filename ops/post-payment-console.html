Post-Payment Console

Use this immediately after Stripe confirms payment so the buyer moves into safe intake, private workspace setup, and scheduling without drift.

Safety Rule

Do not accept production credentials, active tokens, customer data, private source code, or final customer reports in public GitHub issues or ordinary email.

Payment Confirmed Checklist

  1. Confirm package and customer email from Stripe.
  2. Create private customer workspace outside the public repo.
  3. Copy the customer workspace template into the private workspace.
  4. Send Payment Received / Intake Start.
  5. Use a private handoff path before accepting sensitive materials.
  6. Mark the pipeline row as Paid.

Safe Intake Paths

  1. Customer-owned private repo with time-limited read access.
  2. Customer-owned shared folder with time-limited access.
  3. Encrypted archive with password through a separate channel.
  4. Sanitized screenshots or config excerpts.
